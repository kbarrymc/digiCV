Functional Testing

Functional testing will test any functionlity that an end user will interact with on a website or app will work as expected
The test cases we create for this type of testing can be created against the business requirements to ensure that the product being
built meets the needs stated by the business earlier in the SDLC. These requirements give us as QA Analysts, the ability to create Acceptance criteria
as they should clearly define what exactly is being built leaving no room for personal interpretation


Automated Testing

What is automated testing?

Automated testing is when we code the end to end tests or regression tests to run using a coding language of choice and a test framework

Why do we automate our tests?

We automate the tests to speed up the testing process, the initial setup may take time at first but over a longer period this time is then recoperated
due to the speed and capabilities of automated tests

What tests should be automated??

We should only automate tests that will be repeated frequently to reduce effort, time and costs. The most common approach is to automate any regression tests that are performed after every release on the given project
